### 1. Naming
- Use `snake_case` for variables and functions  
- Use `PascalCase` for classes  
- Use clear names like `save_client()` or `load_jobs()`

### 2. File Structure
- Keep each module in its own file  
- Avoid very long files

### 3. Comments
- Comment sections, not every line  
- Explain "why", not "what"

### 4. Functions
- Keep functions short  
- A function should do ONE thing only

### 5. Data Handling
- Always validate JSON data before saving  
- Use try/except only when needed

### 6. Formatting
- Use 4 spaces for indentation  
- Line length under 80–100 characters

### 7. Git
- Write clear commit messages:
  - “Added client creation”
  - “Fixed JSON saving bug”
  - “Implemented job timer”
